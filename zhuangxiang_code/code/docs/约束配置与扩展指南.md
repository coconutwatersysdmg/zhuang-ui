# 装箱约束：配置与扩展指南

本指南面向两类读者：
- **运维/工艺人员**：只想调参数、开关约束 → 看「一、配置使用」。
- **二次开发人员**：想新增约束、改装箱算法 → 看「二、扩展开发」。

---

## 一、配置使用

### 1.1 配置文件在哪

约束统一配置位于 `code/config/packing_config.yaml` 的 `constraints` 段。
运行时默认自动读取该文件；也可用 `--config 路径` 指定其它配置：

```bash
# 用默认配置（code/config/packing_config.yaml）
python run_packing.py

# 用自定义配置
python run_packing.py --config /path/to/my_config.yaml

# 子集快速试参（只装前 1500 箱，便于验证配置效果）
python run_packing.py --max-boxes 1500 --config my_config.yaml --out test.json
```

> 配置加载失败（文件不存在 / 格式错 / 键拼错）会**安全回退到内置默认值**并打印警告，不会中断装箱。

### 1.2 约束清单与配置项

约束分两类：**必须约束**（永远生效，只有数值可调）和 **可关约束**（可通过开关关闭）。

| 约束 | 类型 | 配置项 | 默认 | 说明 |
|---|---|---|---|---|
| 不超界 | 必须·不可关 | —— | —— | 箱子不得超出托盘边界 |
| 不重叠 | 必须·不可关 | —— | —— | 箱子之间不得重叠 |
| 箱间间隙 | 必须·**值可配** | `max_box_gap_mm` | `6.0` | 相邻箱 XY 方向正向间隙必须 < 此值（毫米） |
| 支撑率 | 必须·**值可配** | `support_ratio_threshold` | `0.8` | 非底层箱的直接支撑率必须 ≥ 此值 |
| 重心稳定 | 必须·**值可配** | `center_of_mass_tolerance` | `0.3333…`(1/3) | 重心相对托盘中心的最大允许偏移比例 |
| 吸盘可达 | 可关 | `suction_reachability_enabled` | `true` | 吸盘垂直下放路径不得被遮挡 |
| 小箱在下 | 可关 | `small_box_below_enabled` | `true` | 小箱正下方不得有更大箱（防压坏） |
| 同尺寸重箱在下 | 可关 | `same_size_heavier_below_enabled` | `true` | 同尺寸叠放时重箱在下、轻箱在上 |
| 按倍数凑层 | 可关 | `height_multiple_layering_enabled` | `true` | 同底面不同高度按整数倍优先同层（软偏好） |

吸盘几何（仅 `suction_reachability_enabled=true` 时生效）：

| 配置项 | 默认 | 说明 |
|---|---|---|
| `suction_cup_length` | `600.0` | 吸盘长度（毫米） |
| `suction_cup_width` | `800.0` | 吸盘宽度（毫米） |
| `suction_xy_clearance` | `0.0` | 吸盘 XY 方向安全间隙（毫米） |
| `suction_z_clearance` | `0.0` | 吸盘 Z 方向安全间隙（毫米） |
| `suction_allow_rotation_90` | `true` | 是否允许吸盘旋转 90 度取放 |

### 1.2b 装箱朝向（箱子 90° 旋转）

| 配置项 | 默认 | 说明 |
|---|---|---|
| `allow_box_rotation_90` | `true` | baseline 是否允许箱子 90° 旋转（朝向规整） |

`allow_box_rotation_90` 也位于 `constraints` 段，但它**不是约束**（不拦截放置），
而是 baseline（beam + 配方优先）路径的**装箱朝向能力开关**。开启时，每个箱子在进入
整层规划/落地前，按「托盘上每层格数更大」的朝向规整，并带**几何门槛**：仅对「旋转
能让该箱型单类整层从不达标跨到达标」的箱型换向，对「旋转也救不了的难组」保持原
朝向，避免打乱其已紧凑的布局（保填充率、不增盘数）。这修复了固定朝向对旋转敏感
箱型（如 530×350、740×330）系统性装不满的缺陷。

> GCP 主算法（`main_packer: gcp`）有独立的朝向处理（`_fp_orient` + CP-SAT 90° 旋转），
> **不受此开关影响**；本开关只作用于 baseline 路径（含 GCP 回退的分组）。

### 1.3 常见调整示例

**放宽箱间间隙到 8mm**：
```yaml
constraints:
  max_box_gap_mm: 8.0
```

**关闭机器人吸盘约束**（无机械臂、人工码垛场景）：
```yaml
constraints:
  suction_reachability_enabled: false
```

**换一个更大的吸盘**：
```yaml
constraints:
  suction_cup_length: 700.0
  suction_cup_width: 900.0
```

**关闭小箱在下约束**（确认本批货无重小箱压坏风险时）：
```yaml
constraints:
  small_box_below_enabled: false
```

**关闭箱子旋转**（要求 baseline 固定朝向、应急回退时）：
```yaml
constraints:
  allow_box_rotation_90: false
```

> 改完后建议先用 `--max-boxes 1500` 跑子集，再用 `python tests/verify_solution.py <输出.json>` 核验所有必须约束 0 违例。

### 1.4 重要约束说明

- **必须约束不能关**：不超界、不重叠、间隙、支撑率、重心是装载安全底线，配置里没有对应开关，永远生效；只能调数值。强行在配置里加 `out_of_bounds_enabled: false` 之类无效（未知键被忽略）。
- **放置层与门禁层同源**：每个约束在「装箱时拦截」和「最终门禁校验」两处都按同一份配置生效。关掉某约束时，两处会同时跳过，不会出现「装箱放行但门禁拒绝」的死锁。
- **改约束 = 改结果**：放宽/关闭约束通常会改变装箱方案（达标数、托盘数可能变化）。默认配置（全开 + 标准值）的输出与交付基线**逐托盘完全一致**。

## 二、扩展开发

本节面向需要**新增约束**或**改装箱算法**的二次开发。系统设计了配置驱动 + 约束注册扩展点，绝大多数扩展无需改动核心装箱代码。

### 2.1 架构总览

```
config/packing_config.yaml  (constraints 段)
        │  ConfigLoader.load_constraint_config()
        ▼
ConstraintConfig  (src/config/constraint_config.py)  ← 单一事实来源（frozen dataclass）
        │  run_packing.build_workflow(constraint_config=cfg)
        ├─────────────► PackingWorkflow → PalletPacker  （主装箱）
        ├─────────────► FailedPoolRebuilder / LowFillRepacker /            （救援器类）
        │               TailFragmentAbsorber / LowLoadRebuilder
        ├─────────────► hole_fill / topup / recipe_rebuild（partial 注入）  （救援函数）
        ▼
放置层：BeamSearchPacker / build_direct_layer_packing_solution / PoolCompactor
门禁层：validate_pallet_constraints
        （两层读同一份 ConstraintConfig，保证同源）
```

约束最终落在两个层面：
1. **放置时拦截**（`BeamSearchPacker._generate_feasible_candidates`、`direct_layer_packer._try_pack_pool`）——在生成候选位置时就过滤掉违规放置。
2. **最终门禁**（`validate_pallet_constraints`）——每次装箱/救援后整盘复核，违规则拒绝输出。

### 2.2 新增一个「可配数值」（最简单）

例：给「支撑率」之外再加一个「最小底面接触面积」阈值。

1. 在 `src/config/constraint_config.py` 的 `ConstraintConfig` 加字段：
   ```python
   min_contact_area_mm2: float = 0.0   # 0 表示不限制
   ```
2. 在 `config/packing_config.yaml` 的 `constraints` 段加一行：
   ```yaml
   min_contact_area_mm2: 5000.0
   ```
3. 在需要的约束函数里读 `constraint_config.min_contact_area_mm2`。
   （`from_dict` 会自动认这个新键，YAML 漏写则回退默认 `0.0`。）

完成。无需改 CLI、加载器、入口——`ConstraintConfig` 是 dataclass，新字段自动被 `from_dict`/`to_dict` 接纳。

### 2.3 新增一个「可开关约束」（推荐流程）

例：新增「重箱不得放在轻箱正上方」（跨尺寸的重量约束）。

**第 1 步：写约束谓词**（放 `src/utils/helpers.py` 或新建模块）：
```python
def passes_heavy_not_on_light_constraint(item, point, dims, placed_boxes):
    """重箱不得直接置于更轻箱子之上。True=通过，False=违例。"""
    if point['z'] <= 1e-9:
        return True
    item_weight = float(item.get('weight', 0) or 0)
    for placed in placed_boxes:
        pos = placed.get('position')
        if not pos:
            continue
        placed_top = pos['z'] + placed['height']
        if abs(placed_top - point['z']) > 1e-5:
            continue
        # XY 投影是否重叠（用现成的 helpers._has_positive_xy_overlap）
        if not _has_positive_xy_overlap(point, dims, placed):
            continue
        if item_weight > float(placed.get('weight', 0) or 0) + 1e-9:
            return False
    return True
```

**第 2 步：加配置开关** —— `ConstraintConfig` 加字段 + YAML 加键：
```python
# constraint_config.py
heavy_not_on_light_enabled: bool = True
```
```yaml
# packing_config.yaml
constraints:
  heavy_not_on_light_enabled: true
```

**第 3 步：接入门禁** —— `src/geometry/constraint_validator.py`：
```python
# 顶部 import
from ..utils.helpers import passes_heavy_not_on_light_constraint

# 在 validate_pallet_constraints 解析 config 处加：
heavy_not_on_light_enabled = True
if constraint_config is not None:
    ...
    heavy_not_on_light_enabled = constraint_config.heavy_not_on_light_enabled

# 在逐箱循环里加（参照现有 small_box_below 的写法）：
if heavy_not_on_light_enabled and not (
    passes_heavy_not_on_light_constraint(item, pos, dims, others)
):
    violations.append({"type": "heavy_on_light", "box_id": item_id})
```

**第 4 步：接入放置层** —— 在 `BeamSearchPacker.__init__` 存开关、在
`_generate_feasible_candidates` 候选过滤处加 `if`（参照现有
`small_box_below_enabled` 的两处写法）；`direct_layer_packer.build_direct_layer_packing_solution`
同理。

**第 5 步：写单测**（参照 `tests/test_constraint_config.py` 的
`test_gate_small_box_switch`）：开关开→拦截、关→放行各一条。

**第 6 步：回归**：默认配置（新开关默认 true）跑全量，与基线
`output/NEW_full_only_below.json` 做 `tests/diff_packing.py` 守护差分，
确认新约束不破坏既有达标方案（或按预期变化）。

> 关键原则：**门禁层和放置层必须同时改、用同一个开关**。只改门禁会导致
> 「装箱放行→门禁拒绝→RuntimeError」；只改放置层会导致救援阶段绕过约束。

### 2.4 改装箱算法（替换 packer）

主装箱器通过依赖注入装配，替换无需改 `PalletPacker` 内部逻辑：

- `BeamSearchPacker` 在 `run_packing.build_workflow()` 里作为
  `custom_packer_cls` 注入。要换一个新算法（如 CP-SAT、强化学习 packer），
  实现同样的接口即可：
  - `__init__(self, pallet_dims, ..., constraint_config=None)`
  - `pack(self, items_to_pack, num_restarts, beam_width, candidate_limit,
    random_seed, target_mpm, stop_when_target_met, allow_skip_items)
    -> (placed_boxes, unfitted_items)`
- 放置结果必须带 `position` 和（吸盘开启时）`suction_*` 字段，并能通过
  `validate_pallet_constraints`。
- 在 `build_workflow` 把 `custom_packer_cls=BeamSearchPacker` 换成你的类。

整层确定性装箱（`build_direct_layer_packing_solution`）同样是注入的函数，
可替换为其它整层策略，只要签名兼容（接收 `constraint_config` 参数）。

### 2.5 自检清单（改约束/改算法后必跑）

```bash
# 1. 导入无误
python -c "import run_packing; print('OK')"

# 2. 单测
python tests/test_packing.py
python tests/test_small_box_constraint.py
python tests/test_constraint_config.py
python tests/test_rescue_pool.py

# 3. 子集快速验证 + 必须约束核验
python run_packing.py --max-boxes 1500 --out /tmp/sub.json
python tests/verify_solution.py /tmp/sub.json     # 必须约束应 0 违例

# 4. 全量回归 + 守护差分（确认未破坏基线）
python run_packing.py --out /tmp/full.json
python tests/diff_packing.py output/NEW_full_only_below.json /tmp/full.json --mode guarded
```

## 三、设计要点（维护者备忘）

- `ConstraintConfig`（`src/config/constraint_config.py`）是**唯一事实来源**，frozen
  保证运行期不可变。`from_dict` 容错：未知键忽略、缺省回退默认。
- 注入方式刻意做成「可选参数 `constraint_config=None`」：不传时所有组件退回与
  历史一致的硬编码默认值，所以这次改造是**纯增量、默认行为零变化**（已用全量
  逐托盘等价差分验证）。
- 救援函数（hole_fill/topup/recipe_rebuild）无构造期，用
  `functools.partial(fn, constraint_config=cfg)` 在入口绑定后注入。
- `max_box_gap_mm`（几何间隙，6mm）与救援里的 `max_gap=64.0`（mpm 缺口扫描
  容差）**语义完全不同**，不要混淆——后者不在约束配置内。
